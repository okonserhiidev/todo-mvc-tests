> Источник: `Test task(AQA_SDET)_ TodoMVC (React) – Java + Selenide + JUnit4 + Maven.docx`
> Назначение: оценить умение кандидата спроектировать и реализовать поддерживаемый фреймворк UI-автоматизации.

## 🎯 Цель задания

Построить UI-автоматизацию для эталонного приложения **TodoMVC (React)** и продемонстрировать сеньёрные практики AQA: проектирование, чистый код, надёжные ассерты, репортинг, масштабируемость.

**Целевое приложение:** https://todomvc.com/examples/react/dist/

## 1️⃣ Maven-проект

### Обязательные зависимости в `pom.xml`
- **Selenide**
- **JUnit 4**
- **WebDriverManager** или кастомная настройка драйвера
- **AssertJ** или **Hamcrest** (опционально)
- **SLF4J** для логирования

### Обязательные компоненты фреймворка
- **BaseTest** — единая точка конфигурации браузера, инициализация Selenide
- **Page Objects** — представление страниц / компонентов приложения
- **Test data layer** — enum, константы или провайдер данных
- **Reusable methods** — слой утилит / core для переиспользования

## ✅ Критерии оценки (читаются между строк)

- Архитектурное мышление: чёткие границы слоёв (config / core / pages / data / tests)
- Поддерживаемость: компонентный POM, отсутствие дублирования
- Стабильность: устойчивые селекторы, отсутствие хрупких ожиданий
- Обоснованность выбора: тестовое покрытие должно быть осмысленным, а не «массовым»
- Repeatability: чистое состояние перед каждым тестом
- Качество отчётности: легко читаемые логи, артефакты при падениях
- Готовность к CI/CD: тесты запускаются в pipeline без танцев с бубном
- Готовность к масштабированию: parallel, cross-browser, remote / Selenoid

---

## 🚫 Out of scope (явные не-цели)

- Backend / API тесты — TodoMVC в этом билде работает на `localStorage`, бэкенда нет
- Performance / load testing — другие инструменты (JMeter / k6)
- Visual regression — не требуется
- Mobile responsive — у TodoMVC нет responsive-брейкпоинтов
